# Choisir la prochaine spécification

1. Rechercher les fichiers incomplets dans `docs/ihm/vues` et `docs/ihm/composants`.
2. Proposer un seul fichier prioritaire avec une justification courte.
3. Attendre une validation explicite avant toute modification.
4. Avancer ensuite section par section : proposition, questions nécessaires, validation, écriture.
5. En fin de session, proposer le prochain fichier sans le modifier.

Prioriser les vues cœur de parcours, puis les composants structurants, puis les vues secondaires. Ne pas remplir plusieurs fichiers à la fois et ne pas inventer les informations manquantes.
